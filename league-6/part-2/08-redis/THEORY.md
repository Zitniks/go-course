# Redis

Вы узнаете:
- Что такое кэш и зачем он нужен — почему повторные запросы к БД убивают производительность
- Что такое Redis: in-memory хранилище key-value, работающее в оперативной памяти
- Как Redis хранит данные — пары ключ-значение, поиск за ~100 наносекунд, O(1) по хэшу
- Структуры данных Redis: строки (String), списки (List), хэши (Hash), множества (Set), сортированные множества (ZSet)
- TTL — как ограничивать время жизни ключей, чтобы кэш не протухал и память не кончалась
- Типичные паттерны применения: кэш профилей, сессии, rate limiting, лидерборды (ZSet)
- Репликация Redis: мастер + слейвы, eventual consistency при асинхронном копировании
- Горизонтальное масштабирование: шардирование через консистентное хэширование
- Проблема race condition при параллельном обновлении и атомарные команды Redis (`INCR`, `GETSET`)
- Чем Redis отличается от обычных NoSQL-баз: нет JOIN-ов, нет сложных запросов, только ключ
- Когда Redis не подходит: сложная фильтрация, нужна strong consistency (например, баланс счёта)
