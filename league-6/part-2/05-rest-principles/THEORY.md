# REST API — принципы проектирования

Вы узнаете:
- Что такое REST и какие у него ключевые принципы (stateless, client-server, uniform interface)
- Как называть эндпоинты через ресурсы, а не действия (`/users`, а не `/getUsers`)
- Как правильно использовать HTTP-методы: GET, POST, PUT, PATCH, DELETE
- Вложенные ресурсы и когда их применять (`/users/{id}/orders`)
- Какие HTTP-коды отвечают каким ситуациям: 200, 201, 204, 400, 401, 403, 404, 409, 500
- Формат тела запроса и ответа: JSON, заголовки Content-Type / Accept
- Версионирование API (`/api/v1/...`) и зачем оно нужно
