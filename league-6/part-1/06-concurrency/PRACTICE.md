# Практика — Платёжный модуль и конкурентный обработчик

---

## Шаг 1 — Интерфейс способа оплаты

Объяви интерфейс `PaymentMethod` с двумя методами:
- `Pay(usd int) int` — проводит оплату, возвращает ID операции
- `Cancel(id int)` — отменяет операцию

Реализуй две заглушки: `CryptoPayment` и `CardPayment`. Каждая выводит своё сообщение и возвращает случайный ID.

---

## Шаг 2 — Платёжный модуль

Создай структуру `PaymentModule` с полем `method PaymentMethod`.

Добавь методы:
- `Pay(description string, usd int) int` — проводит оплату через `method`, сохраняет информацию об операции, возвращает ID
- `Cancel(id int)` — отменяет операцию и помечает её как отменённую
- `Info(id int) PaymentInfo` — возвращает информацию об операции
- `AllInfo() map[int]PaymentInfo` — возвращает копию всех операций

Структура `PaymentInfo` должна содержать: описание, сумму и флаг отмены.

> Подумай, почему для хранения операций лучше подходит мапа, а не слайс.

---

## Шаг 3 — Обработка ошибок

Добавь в метод `Pay` проверку: если `usd <= 0` — возвращать ошибку вместо ID.

Измени сигнатуру метода на `Pay(description string, usd int) (int, error)`.

В `main` обработай эту ошибку и выведи понятное сообщение.

---

## Шаг 4 — Конкурентный обработчик заказов

Напиши функцию `ProcessOrders(orders []Order, method PaymentMethod) []int`, которая обрабатывает список заказов параллельно.

Каждый заказ — отдельная горутина. Результаты (ID операций) собери через канал. Дождись завершения всех горутин через `WaitGroup`.
```go
type Order struct {
    Description string
    Amount      int
}
```

---

## Шаг 5 — Контекст и отмена

Добавь в `ProcessOrders` поддержку контекста: `ProcessOrders(ctx context.Context, orders []Order, ...)`.

Если контекст отменяется до начала обработки очередного заказа — горутина должна завершиться без оплаты и вывести сообщение об отмене.

В `main` отмени контекст через 2 секунды и передай список из 10 заказов с задержкой 500мс каждый.
